# MVP Scope Document

## Overview
This document outlines the minimum viable product scope for the MiniiERP system, defining core modules, features, and user roles required for the initial release.

## Core Modules

### 1. Product Management
- Product catalog creation and management
- Basic product information (name, description, price, category)
- Stock level tracking (basic quantities)
- Product search and filtering capabilities

### 2. Procurement
- Supplier information management
- Purchase order creation and tracking
- Basic procurement workflow (request → approve → receive)
- Purchase history tracking

### 3. Sales
- Customer information management
- Sales order creation and processing
- Basic sales workflow (quote → order → invoice)
- Sales history tracking

### 4. Dashboard
- Overview of key metrics (inventory levels, recent orders, pending tasks)
- Quick access to core functionalities
- User-specific dashboards based on role

## Primary User Journey
The MVP must support at least one complete user journey end-to-end:
1. A user creates a product
2. A user places a purchase order for that product
3. A user processes a sales order for that product
4. A user views dashboard metrics

## Out of Scope Features
- Advanced reporting features
- Complex inventory tracking beyond basic stock levels
- Multi-user concurrent access controls
- Integration with external systems

## Acceptance Criteria
- All four main modules are included in the scope
- Core features required for primary journey are identified
- Technical decisions are clearly defined and agreed upon
- User roles and their permissions are documented
